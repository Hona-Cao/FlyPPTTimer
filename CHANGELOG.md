# CHANGELOG

## 0.13.0

- 新建 `FlyPPTTimer.Core`、`FlyPPTTimer.Infrastructure.Windows`、`FlyPPTTimer.WinUI`，以 .NET 8、WinUI 3、Windows App SDK 稳定版和 CommunityToolkit.Mvvm 重构桌面程序。
- 主窗口改用 Mica、Fluent 卡片和左侧 NavigationView；新增概览、计时、PowerPoint、文件规则、手机遥控、外观、快捷键、设置及诊断页面。
- 计时悬浮窗改为独立 WinUI/Win32 窗口，支持无边框、透明、置顶、穿透、锁定、多屏、九宫格位置及独立文字/背景透明度。
- PowerPoint COM 全部保留在专用 STA 有限队列；新增首张幻灯片异步缩略图缓存，并保持计时和远控在 COM 异常时可用。
- 文件规则新增多选、清空确认、拖放添加和排序、按钮排序、批量时长、状态标签、重新定位与详情预览。
- 手机演示列表同步桌面顺序、目录、时长和状态，并提供打开、切换、从头及当前页放映。
- 恢复提示、蜂鸣、语音、自定义 WAV、静音、暂停闪烁及文字/背景/边框独立闪烁入口。
- 新增 WinUI 构建与独立发布脚本、Windows CI，以及 v0.13 Core 自动化回归。

## 0.12.1

- PowerPoint 缓存事件和全屏检测统一接入放映生命周期控制器，完整遵守 `AutoStartOnFullscreen` 并去重。
- 修复手机演示列表在命令 busy 状态后永久禁用；区分当前、切换和打开，并显示同名文件目录。
- 计时按钮按运行、暂停、停止和结束状态限制操作；跳页增加最大页数和前端边界校验。
- 远控 token 改为固定时间比较，日志移除查询参数；响应增加 CSP、Referrer-Policy 和 nosniff，移除通配 CORS。
- “断开所有设备”会重新生成 token、清空设备状态，使旧链接立即失效。
- 配置保存改为临时文件验证、原子替换和最近五份备份恢复；安装升级备份且不覆盖用户配置。
- 增加单实例保护、全局异常捕获、日志轮转、PowerPoint 刷新错误节流和 STA 安全退出。
- 增加 17 项 xUnit 自动化测试及 Windows GitHub Actions；PowerPoint COM 操作标记为本机集成测试。
- 发布脚本继续从 csproj 读取版本并拒绝覆盖已有版本目录。

## 0.12.0

- 设置窗口最小化、最大化、关闭按钮由 `34×30` 放大为 `68×58`，标题栏同步增高。
- PowerPoint COM 全部收口到 `PowerPointControlService` STA 线程；全屏检测只读取缓存状态。
- STA 服务增加 32 项有限队列、命令超时、协作取消、COM 忙碌重试和后台刷新合并。
- `/state` 改读缓存的 `PresentationState`，PowerPoint 异常不再阻塞计时遥控连接。
- HTTP 服务按 `Content-Length` 字节读取，限制请求头、请求体、并发连接，并设置收发与连接超时。
- 手机网页改为串行轮询，命令期间禁止重复点击；增加翻页防抖、黑白屏选中态、结束确认和 `focus-visible`。
- 手机端直接滚动列出全部允许的演示文稿，移除 PPT 下拉选择框。
- Clash/TUN/代理虚拟网卡降级为不可推荐地址，优先真实 Wi-Fi、以太网和私有局域网 IP。
- 修复计时结束快照顺序，以及退出放映时停止、重置、已结束状态的组合处理。
- 放映文件名、路径、页码和总页数统一读取同一个 `SlideShowWindow.Presentation`。
- 发布脚本从 csproj 读取版本并拒绝覆盖已存在的版本目录。

## 0.11.0

- 在现有远控端口、token、二维码和网页中加入 PowerPoint 演示控制，不创建第二套服务。
- 新增 `PowerPointControlService`，所有 PowerPoint COM 调用在专用 STA 线程中串行执行并释放 COM 对象。
- `/state` 同时返回 `timerState` 和 `presentationState`，`/command` 新增放映、翻页、跳页、黑白屏、结束和白名单文件选择命令。
- 手机网页拆分为独立嵌入式 HTML、CSS、JavaScript 资源，并重做为计时/演示双页签移动端卡片界面。
- 文件选择仅接受当前已打开文稿或已启用文件规则对应的服务端 ID，拒绝任意路径和未知命令。
- 远程启动放映与原全屏检测共用去重后的计时联动入口；开始时匹配文件规则并启动计时，结束后停止重置。
- 修复 PowerPoint“从当前页放映”必须设置指定幻灯片范围的问题，并忽略连续快速重复启动。
- 新建独立版本目录 `releases\v0.11`，保留所有旧版本目录。

## 0.10.0

- 修复设置窗口顶部导航栏仍出现原生黑色竖线、硬边框和阴影残留的问题。
- 移除设置窗口导航区的 WinForms 原生 `TabControl` 页签头，改为完全自绘色块导航按钮和独立内容容器。
- 保留原设置页切换、滚动和所有设置项功能；本轮不修改远程控制、右键菜单、PPT 自动启动计时或文件规则逻辑。
- 更新独立发布目录 `releases\v0.10`，保留 v0.9 目录。

## 0.9.0

- 修复设置窗口外缘仍残留系统矩形边框、矩形阴影或方形外壳的问题；设置窗口改为无系统边框的自绘圆角窗体。
- 为设置窗口补充自绘标题栏、最小化、最大化、关闭、标题栏拖动和边缘缩放，保留可调整大小体验。
- 全局收小设置窗口内外圆角半径约 30%，包括页签、按钮、输入框、下拉框、分组标题条、表格、卡片和内容容器。
- 保持本轮仅外观改动，不修改远程控制、右键菜单、PPT 自动启动计时和文件规则逻辑。
- 更新独立发布目录 `releases\v0.9`，保留 v0.8 目录。

## 0.8.0

- 优化远程控制设置页排版，统一行高、边距、说明行和按钮尺寸，并明确端口生效规则。
- 调整远程控制端口策略：服务运行中不因普通“应用”自动重启或换端口，修改端口需点击“重启远程服务并应用端口”或下次启动后生效。
- 默认计时器窗口位置改为“上中”，垂直微调默认 `0.5%`；未设置过位置的旧配置会补齐该默认值。
- 重做“文件规则”模块：支持添加多个 PPT 文件、删除、清空、表格查看和选中项编辑；规则包含文件名、文件路径、计时时长和启用状态。
- PowerPoint 放映开始时会读取当前放映文件路径，优先匹配文件规则时长；未匹配时继续使用默认时长。
- 设置窗口顶部导航栏改为胶囊式页签，取消硬边框和黑色竖线观感，增加页签间距与内容区间距。
- 更新独立发布目录 `releases\v0.8`，保留 v0.7 目录。

## 0.7.0

- 修复手机端远程开始后电脑端计时器窗口不实时刷新的问题；远程命令现在统一调度到 WinForms UI 线程执行。
- 手机端新增“已连接/连接中/已断开”和最后同步时间显示，断开时禁用控制按钮并提示重连。
- 梳理手机端暂停/停止语义：暂停保留当前时间，继续从暂停处恢复；停止改为停止并重置到默认时长。
- 新增设置项“暂停时闪烁当前时间”，默认关闭。
- 删除手机端“测试提示”按钮，并移除远程 `prompt.test` 命令入口。
- 修复手机端“触发闪烁”不驱动电脑端窗口的问题，统一走本地闪烁逻辑。
- 再次修正托盘和计时器窗口右键菜单位置：按菜单实际宽高计算，右下角贴近鼠标指针，并限制在当前屏幕工作区内。
- PowerPoint/WPS 放映开始优先级提升：进入放映时无论此前状态如何，一律重置为默认时长并立即开始计时。
- 新增独立发布目录 `releases\v0.7`，保留历史版本目录。

## 0.6.0

- 计时器窗口和托盘右键菜单统一按鼠标指针右下角对齐弹出，并支持点击空白区域关闭。

## 0.5.0

- 设置窗口改为可拖拽调整大小，分页滚动由 TabPage 接管，修复滚动到底仍有内容露不全的问题。
- 设置窗口、常用输入控件、按钮、分组条和右键菜单改为小圆角外观。
- 底部“应用、取消、确定”按钮加高并固定在窗口底部，避免中文下半部分被截断。
- 合并“外观设置”“显示位置”“提示样式”为“外观与显示”，删除不再需要的额外遥控设置入口。
- “其他设置”改为统一表单布局，避免横向挤压和横向滚动条。
- 默认悬浮窗口改为 200×60，默认字号改为 20。
- 计时器默认显示分秒，如 `08:00`；需要小时时自动显示 `00:00:00`。
- PowerPoint 放映检测新增 COM 路径，识别到放映窗口后自动开始计时。
- WPS 演示和其他白名单程序改为扫描所有可见顶层窗口，不再只依赖前台窗口。
- 自动因放映开始的计时，在退出放映后强制停止并重置，下一次放映重新从默认时长开始。
- 去掉自绘控件内部边线，按钮、输入框、下拉框、分组条改为色块式小圆角。
- 数字参数改为普通数字输入框，下拉框和参数控件禁用鼠标滚轮误操作。
- 远程控制改为默认开启，并在计时器/托盘右键菜单中增加二维码入口。
- 托盘图标右键菜单改为按当前鼠标位置手动弹出。
- 修复手机网页遥控状态字段大小写不一致导致按钮看似无反应的问题。
- 手机遥控页面改为全中文可读文本，并保留 HTTP polling 自动重连。
- 明确截图验收方法：高 DPI 屏幕截图前启用 DPI 感知，主屏截图尺寸应为 2560×1600。

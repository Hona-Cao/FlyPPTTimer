# CHANGELOG

## 0.20.0

- “其他设置”新增“启动时检测新版本”，默认关闭；用户启用并保存后，下次启动会从 Gitee Release 检测稳定更新。
- 系统托盘右键菜单新增“检测新版本”，可随时手动检查；无 Release、已是最新版和网络错误均提供明确反馈。
- 检测到新版本时展示版本号和更新说明并由用户确认。安装版会下载 Gitee Release 中的 Windows x64 setup，若同时发布 SHA-256 文件则先校验，主程序退出后再覆盖安装，并沿用安装器现有的配置保留机制。
- 自动识别绿色便携版；便携版不自动覆盖程序目录，确认后打开对应的 Gitee Release 页面，由用户自行选择下载文件。
- 新增 Gitee API、版本比较、附件选择、默认关闭策略及安装/便携分流的自动化回归测试。

## 0.19.2

- 手机页面滑动不再限定起手区域或水平夹角；页面内容、按钮和输入区域均可识别有效左右位移，并在拖动后抑制误点击。
- 滑动触发阈值降低并加入速度判断，提高短距离和斜向手势的响应灵敏度。
- 切换动画改为可抢占：新手势或标签点击会读取页面当前可见位移、立即冻结上一段动画，再从该位置执行新方向，支持连续反向操作覆盖尚未结束的切换。

## 0.19.1

- 手机“计时 / 演示”切换改为跟手拖动，松手后以平滑动画吸附到目标页面，并随当前页面内容调整可视区域高度。
- 修复“从头放映 / 从当前页放映”写入 `SlideShowSettings` 后使外部打开文稿进入已修改状态的问题：启动前保存范围设置，启动后立即恢复；仅当文稿原本无未保存修改时恢复干净状态，避免掩盖真实用户编辑。
- 计时窗口和托盘图标的右键菜单统一精简为“重置计时窗口位置、静音/取消静音、远程控制、设置、退出”。

## 0.19.0

- 修复精简危险操作按钮时误把 PC 远程控制外层布局从两列改成一列的问题；侧栏和主工作区恢复左右排列，并增加布局回归测试。
- 默认计时窗口调整为 100×35，继续沿用固定中心原点和单行水平、垂直居中规则。
- 手机遥控的“计时”和“演示”页面支持左右滑动切换，同时保留标签点击切换。
- 外部打开的 PowerPoint 文稿也可通过遥控关闭；每次按打开顺序关闭最后一个文稿，关闭时不弹出保存确认，直至没有文稿后按钮才不可用。
- 删除功能重复的“退出 PowerPoint”，将强制退出入口统一简化为“退出演示软件”。
- PC 或手机修改全局默认时长且已有文件规则时，明确询问是否同步全部规则；不同步时保留各文稿原有时长。
- “其他设置”同时提供 GitHub 主仓库和中国大陆可直接访问的 Gitee 镜像，并补充作者目前就职于江苏省人民医院宿迁医院。

## 0.18.9

- 修复计时窗口在句柄创建前设置物理尺寸、随后被 WinForms 再次执行创建期 DPI 缩放的问题；现在先创建目标显示器上的 Per-Monitor V2 窗口句柄，再设置最终尺寸和中心位置，窗口缩放不再只向右、向下增长。
- 明确“默认点位 + 水平/垂直微调”为唯一位置原点；窗口尺寸变化始终用最终宽高重新计算左上角，使窗口中心保持在该原点，最多只有像素取整造成的 0.5 像素误差。
- 时间绘制改为与形状内文本相同的单行水平居中、垂直居中，不再使用独立的冒号/分钟锚点算法。

## 0.18.8

- 计时文字改为按实际可见字形边界定位；100×30 等小尺寸下仍按指定时间锚点水平居中，并按字形中心垂直居中，不再出现文字贴底或偏右。
- 手机“计时”和“演示”页面始终显示“时间到”黑屏控制；无黑屏时明确显示不可用，黑屏出现后可直接退出。
- 针对 WPS 的 `PP12FrameClass` 外层演示窗口，在首次显示事件中立即最大化，并在打开后再次校验；不再依赖 WPS 无效的 PowerPoint COM 主窗口句柄。

## 0.18.7

- 首次针对 100×30 等紧凑计时窗口改用可见字形度量，以时间文字的可见边界计算水平、垂直中心，减少小尺寸下的贴底和偏移。
- 手机端“计时”和“演示”两个页面都加入退出“时间到”黑屏的控制入口，并把黑屏是否存在同步到网页状态。
- 为 WPS 增加外层演示窗口识别与首次显示最大化处理，避免只修改兼容层 COM 窗口状态而实际界面仍以小窗口出现。
- 增加紧凑计时文字、双页面黑屏按钮和 WPS 外层窗口最大化的自动化契约测试；后续 v0.18.8、v0.18.9 在真实设备验证基础上继续加固这些实现。

## 0.18.6

- WPS/PowerPoint 打开文稿时在首个可见帧之前预设最大化状态，避免先显示普通窗口再最大化的跳变。
- 将设置中的默认屏幕、点位和偏移定义为固定中心原点；重置位置回到该原点，宽度、高度及字体变化都围绕原点向四周对称展开。
- 移除计时窗口 40 像素最小高度限制，宽高只在文字实际放不下时按需扩展。
- 手机端修改计时方式或时长时同步写入当前演示文稿的文件规则，后续从手机或电脑开始放映都会使用新设置。
- 手机端“计时”和“演示”页面均增加退出“时间到”黑屏按钮，并实时同步黑屏状态。

## 0.18.5

- 删除文件规则列表中重复的“规则已启用”状态块，只保留可操作的启用/禁用按钮；切换状态和选择规则时保持当前滚动位置。
- 文件规则新增复选框和批量设置，可一次修改多条规则的时长及正计时/倒计时方式，并在放映开始时同步应用。
- 取消计时窗口人为设置的 80 像素最小宽度；仅在文字确实无法完整显示时围绕默认点位自动扩宽。
- 计时字体固定为微软雅黑并从设置页移除；删除尚未开发的单页计时设置。
- 黑屏到时动作会先结束后台放映并停止、重置计时器；只有“仅提示”允许继续显示超时。
- 由 FlyPPTTimer 打开的 PowerPoint 文稿继续使用只读方式，并默认激活为最大化窗口。

## 0.18.4

- 修复 v0.18.3 仍会在每次重建悬浮窗时由 WinForms 执行第二次 DPI 自动缩放、且只保持左上角不变的问题；现在先按目标显示器 DPI 显式计算最终物理尺寸，再计算中心位置，并禁用显示阶段的二次缩放。
- 修复第一次应用设置后，设置窗口与主程序共用同一个配置对象的问题；后续修改默认点位或偏移现在能正确识别为位置变化。
- 完整设置链路验证：修改时长前后 Bounds 完全一致；修改尺寸围绕原中心展开；修改垂直偏移后窗口按新位置移动。
